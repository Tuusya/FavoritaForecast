# Прогнозирование продаж для сети магазинов Favorita

## Описание задачи
Цель проекта — разработать модель для прогнозирования ежедневных продаж в магазинах Favorita на основе исторических данных и внешних факторов. Задача решается в рамках соревнования на Kaggle. Основная метрика качества — **RMSLE** (Root Mean Squared Logarithmic Error).

## Данные
В работе использовался следующий набор данных соревнования, включающий:
- train.csv — история продаж (около 4.2 млн записей после фильтрации)
- items.csv, stores.csv — статические характеристики товаров и магазинов
- holidays_events.csv — информация о праздниках
- oil.csv — ежедневные цены на нефть
- transactions.csv — количество транзакций по магазинам

Из-за большого объёма данных и ограничений по памяти часть экспериментов проводилась в **Google Colab**, а для обучения модели TFT потребовалось использовать **Kaggle Notebooks**. В репозитории представлены два блокнота: 
- `notebooks/ts_hw3.ipynb` — все эксперименты с бейзлайнами и CatBoost.
- `notebooks/tft.ipynb` — обучение Temporal Fusion Transformer и анализ результатов.
Отчет о выполненой работе находится в файле report.pdf

## Инструкция по воспроизведению
1. Клонировать репозиторий.
Для ноутбука Kaggle
2.В Session option выбрать GPU
3. Выбрать Favorita в разделе Input, через кнопку Add input.
4. Отдельно загрузить train.csv, через кнопку Upload.
5. Указать путь к загруженному файлу в ячейке с переменной path_to_train
6. Запустить блокнот.
Для ноутбука Colab
7.Выбрать среду выполнение на GPU
8. Загрузить файлы в формате csv, кроме train
9. Загрузить train.csv.7z
10. Запустить блокнот
